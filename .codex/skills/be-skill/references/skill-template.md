# SKILL.md Template

用于生成新 skill 的主文档骨架。请将占位符替换为真实内容，不保留示例文本。

```md
---
name: {skill-name}
description: {一句话说明技能用途、触发场景和价值}
---

# {skill-name}

## 何时使用

- {触发条件 1}
- {触发条件 2}

## 输入前置清单

- {目标能力}
- {输入输出边界}
- {环境依赖}
- {验收标准}

## 执行流程

### 1) 收集上下文

- {读取哪些文件、命令或日志}
- {如何识别关键实现}

### 2) 设计结构

- {目录结构（固定输出到 skills/{skill-name}）}
- {模块边界}

### 3) 实施步骤

- {步骤 1，含验证方式}
- {步骤 2，含验证方式}
- {步骤 3，含验证方式}

### 4) 错误处理

- {常见失败场景}
- {回退/修复策略}

### 5) 输出与验收

- {交付物列表}
- {验证命令，例如: bash .codex/skills/be-skill/scripts/quick-check.sh <skill-name>}
- {完成判定}

## 执行约束

- 不引用 skill 目录外路径。
- 生成结果固定写入项目根目录 `skills/{skill-name}`。
- 功能/接口类 `references/*.md` 文档至少包含一段 fenced code block 代码示例。
- 官方文档采用“一个依赖一个 md”方式放在 `references/official-docs/`。
- `references/official-docs-fallback.md` 维护 required/optional 索引路径，并与实体文件一致。
- 不写死依赖名称；按当前技能实际依赖生成 `references/official-docs/*.md`。
- 优先复用官方 SDK / 主流工具，不重复造轮子。
- 若启用依赖第三方 API 凭据（如 `apiKey`）的能力，必须反问并收集对应凭据；缺失时停止该能力实现。
- 凡是“必须由用户提供”的信息，反问时必须同步给出获取建议或文档链接（至少一个，优先 skill 内路径）。
- 凡是“必须由用户提供”的信息，缺失时必须暂停全部后续流程并等待用户输入后再继续；禁止猜测填充或临时兜底。
- 启用依赖第三方 API 凭据的能力时，禁止使用 mock/console/空实现作为默认兜底路径。
- 若交付包含新增页面或组件，必须与原项目视觉风格保持一致，并复用既有样式体系。
- 发现需求冲突先停止并升级说明。
```
