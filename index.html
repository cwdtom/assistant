快速开始，不需要前端堆栈。

仓库本身是一个 Python CLI 项目，GitHub Pages 只负责展示说明页。你可以直接使用现有脚本完成初始化、配置与运行。

01

初始化环境

推荐直接使用一键脚本创建虚拟环境、安装依赖并初始化数据库。

./scripts/bootstrap.sh ./scripts/bootstrap.sh --dev
02

填充配置

复制 `.env.example` 为 `.env`，至少填入 `DEEPSEEK_API_KEY`。模型与运行开关都通过环境变量控制。

cp .env.example .env
03

启动 Assistant

可以直接运行入口，也可以使用附带脚本管理后台实例、状态与重启。

python main.py ./scripts/assistant.sh start ./scripts/assistant.sh status

保留明确命令，也支持自然语言驱动。

你可以像传统 CLI 一样精确执行命令，也可以直接输入目标。当前 MVP 已覆盖帮助、版本、日期、画像刷新、主动提醒、日程、历史与碎片想法管理。

基础入口

用最小命令确认环境是否正常，再逐步进入任务流。

/help
/version
/date
/profile refresh
/notify

结构化管理

对于日程、历史和碎片想法，命令接口保持稳定，适合直接脚本化或手工操作。

/history search 会议
/thoughts list
/schedule repeat <id> on
